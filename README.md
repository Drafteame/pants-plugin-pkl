# pants-plugin-pkl

A [Pants](https://www.pantsbuild.org) plugin for the [PKL configuration language](https://pkl-lang.org). It brings PKL files into the Pants build graph, enabling `pants test`, `pants fmt`, `pants lint`, `pants package`, `pants tailor`, and automatic dependency inference for `.pkl` files.

## What this plugin does

`pants-plugin-pkl` integrates PKL into a Pants monorepo so that `.pkl` source files are first-class build targets. The plugin downloads the `pkl` binary automatically (no manual installation required) and uses it to:

- **Validate** PKL files compile cleanly (`pants lint` via `pkl eval`)
- **Format** PKL files in-place or check formatting in CI (`pants fmt` / `pants lint` via `pkl format`)
- **Run tests** written with `pkl:test` (`pants test`)
- **Package** PKL modules into JSON, YAML, XML, and other formats (`pants package`)
- **Suggest BUILD targets** automatically for new `.pkl` files (`pants tailor`)
- **Infer dependencies** between `.pkl` files so you never have to list them by hand

## Installation

1. Copy the `pants-plugins/pkl/` directory into your repository.

2. Add the plugin directory to `pythonpath` and declare the backends in `pants.toml`:

```toml
[GLOBAL]
pythonpath = ["%(buildroot)s/pants-plugins"]
backend_packages.add = [
  "pants.backend.plugin_development",
  "pkl",
  "pkl.goals",
  "pkl.lint.fmt",
  "pkl.lint.eval_check",
]
```

3. Add `pants-plugins` to source roots so Pants can find the plugin modules:

```toml
[source]
root_patterns = ["pants-plugins", "/"]
```

4. Add a `pants_requirements` target in `pants-plugins/BUILD` so Pants can resolve the plugin's own dependencies:

```python
pants_requirements(name="pants")
```

5. Configure the Python resolve for the plugin:

```toml
[python]
enable_resolves = true

[python.resolves]
pants-plugins = "pants-plugins/lock.txt"
```

## Quick start

Given a directory with some `.pkl` files:

```
src/
  config.pkl
  config_test.pkl   # amends "pkl:test"
```

Run `pants tailor src::` to generate BUILD entries automatically:

```python
# src/BUILD  (generated by pants tailor)
pkl_sources(name="src")
pkl_tests(name="tests")
```

Then use Pants as normal:

```bash
# Validate all PKL files compile
pants lint src::

# Format PKL files (requires PKL >= 0.30.0)
pants fmt src::

# Run PKL tests
pants test src::

# Package a module to JSON
pants package src:config
```

To package a module, declare a `pkl_package` target explicitly:

```python
# src/BUILD
pkl_package(
    name="config-pkg",
    source="config.pkl",
    output_format="json",
)
```

## Configuration

All options can be set in `pants.toml` or passed on the command line.

### `[pkl]` — Tool

Controls which PKL binary version is downloaded and used by all goals.

```toml
[pkl]
version = "0.31.0"
```

| Option | Default | Description |
|---|---|---|
| `version` | `"0.31.0"` | PKL CLI version to download |
| `known_versions` | (built-in) | SHA-256 checksums and sizes for each platform binary |

### `[pkl-fmt]` — Formatter

```toml
[pkl-fmt]
skip = false
args = []
```

| Option | Default | Description |
|---|---|---|
| `skip` | `false` | Skip formatting entirely when running `pants fmt` / `pants lint` |
| `args` | `[]` | Extra arguments forwarded to `pkl format` (e.g. `["--grammar-version", "1"]`) |

> Requires PKL >= 0.30.0. The `pkl format` subcommand was introduced in that release.

### `[pkl-eval-check]` — Eval lint check

```toml
[pkl-eval-check]
skip = false
args = []
```

| Option | Default | Description |
|---|---|---|
| `skip` | `false` | Skip the eval check when running `pants lint` |
| `args` | `[]` | Extra arguments forwarded to `pkl eval` (e.g. `["--no-project"]`) |

### `[pkl-test-runner]` — Test runner

```toml
[pkl-test-runner]
skip = false
args = []
timeout_default = 0
overwrite = false
```

| Option | Default | Description |
|---|---|---|
| `skip` | `false` | Skip all PKL tests when running `pants test` |
| `args` | `[]` | Extra arguments forwarded to `pkl test` |
| `timeout_default` | `0` | Default timeout in seconds (`0` means no timeout); overridden per-target by the `timeout` field |
| `overwrite` | `false` | Pass `--overwrite` to regenerate `.pkl-expected.pcf` snapshot files |

## Target types

### `pkl_source`

A single PKL source file.

```python
pkl_source(
    name="config",
    source="config.pkl",
    dependencies=[],          # explicit deps (usually inferred automatically)
    project_dir=None,         # path to PklProject directory
    skip_eval_check=False,    # skip lint eval check for this target
)
```

### `pkl_sources`

Generates one `pkl_source` target per matching file. Excludes test files and `PklProject` by default.

```python
pkl_sources(
    name="src",
    sources=["*.pkl", "!*_test.pkl", "!*Test.pkl", "!test_*.pkl", "!PklProject"],
)
```

### `pkl_test`

A single PKL test module (must `amends "pkl:test"`).

```python
pkl_test(
    name="math-test",
    source="math_test.pkl",
    timeout=30,               # seconds; overrides [pkl-test-runner].timeout_default
    junit_reports=False,      # produce JUnit XML in .junit/
    skip_test=False,
)
```

### `pkl_tests`

Generates one `pkl_test` target per matching test file.

```python
pkl_tests(
    name="tests",
    sources=["*_test.pkl", "*Test.pkl", "test_*.pkl"],
)
```

### `pkl_package`

Evaluates a PKL module and writes the output to `dist/`. Supports single-file, multi-file, and expression modes.

```python
pkl_package(
    name="config-pkg",
    source="config.pkl",
    output_format="json",     # json | yaml | plist | properties | pcf | textproto | xml | jsonnet
    output_path=None,         # defaults to <stem>.<ext> (e.g. config.json)
    multiple_outputs=False,   # use output.files mechanism for multi-file output
    multiple_output_path=".", # base directory for multi-file output
    expression=None,          # evaluate a sub-expression instead of the whole module
    project_dir=None,
    module_path=None,         # directories/archives to search for modulepath: URIs
    extra_args=[],
)
```

## Goals

### `pants test`

Runs `pkl test <source>` for each `pkl_test` target. Exit codes: `0` = passed, `1` = failed, `10` = expected snapshot files written.

```bash
pants test src::
pants test src:math-test
```

Snapshot files (`.pkl-expected.pcf`) in the same directory are automatically included in the sandbox.

### `pants fmt`

Runs `pkl format --write` on all `pkl_source` and `pkl_sources` targets. Requires PKL >= 0.30.0.

```bash
pants fmt src::
```

### `pants lint`

Runs two independent checks:

- **`pkl-eval-check`**: runs `pkl eval --format json -o /dev/null` to verify each source file compiles. JSON format is used instead of the default PCF renderer because PCF cannot serialize `Map` values or certain PKL-typed objects. Targets with `skip_eval_check=True` are excluded.
- **`pkl-fmt`**: checks that files are already formatted (runs `pkl format` and compares output).

```bash
pants lint src::
pants lint --only=pkl-eval-check src::
pants lint --only=pkl-fmt src::
```

### `pants package`

Runs `pkl eval` and writes the output to `dist/`. Supports JSON, YAML, XML, and all other PKL output formats.

```bash
pants package src:config-pkg
```

### `pants tailor`

Automatically suggests `pkl_sources()` and `pkl_tests()` BUILD entries for unowned `.pkl` files. Test files are detected by reading the file header for `amends "pkl:test"`. `PklProject` files are always excluded.

```bash
pants tailor src::
```

### `pants dependencies`

Dependencies between `.pkl` files are inferred automatically using `pkl analyze imports`. You generally do not need to list `dependencies` manually.

```bash
pants dependencies src:config
```

## External package dependencies

PKL projects that use external packages (via `PklProject` dependencies like `@formae` or `package://` URIs) require those packages to be **vendored** into a `pkl-packages/` directory at the repository root. This is because the Pants sandbox restricts network access during builds.

To set this up:

1. Make sure your `PklProject` and `PklProject.deps.json` files are committed to the repository.

2. Run `pkl project resolve` locally to download packages into `pkl-packages/`:

```bash
pkl project resolve -o pkl-packages/
```

3. Commit the `pkl-packages/` directory to your repository (or generate it in CI before running Pants).

4. Set `project_dir` on your targets if the `PklProject` file is not in the same directory as the PKL source:

```python
pkl_sources(
    overrides={
        "config.pkl": {"project_dir": "config"},
    },
)
```

The plugin automatically includes all `PklProject`, `PklProject.deps.json`, and `pkl-packages/**` files in the sandbox for `eval-check`, `test`, `package`, and dependency inference goals.

> **Note:** `pants fmt` does not need external packages because `pkl format` does not resolve imports.

## PKL version requirements

| Feature | Minimum PKL version |
|---|---|
| `pants test`, `pants lint` (eval-check), `pants package` | 0.27.0 |
| `pants fmt` (`pkl format`) | 0.30.0 |
| Default / recommended | **0.31.0** |

> **Note:** The plugin ships with pre-computed SHA-256 checksums only for version **0.31.0** (macOS arm64/x86_64, Linux arm64/x86_64). To use a different version, you must provide the checksums yourself via the `[pkl].known_versions` option in `pants.toml`.
